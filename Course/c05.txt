GUI elements measurements:
UI design: don't use px (pixels), don't use pt (points). Instead use: 
-dp (density-independent pixels) - to scale GUI elements according to current machine running the app
-sp (scale-independent pixels) - applied on fonts only

scaling:
ldpi 0.5
mdpi 1.0
hdpi 1.5
xhdpi 2.0
xxhdpi 3.0
xxxhdpi 4.0
dh = px*160/dpi
px = dp*dpi/160

scaling images ratio rule:
3:4:6:8:12:16

36x36 ldpi
48x48 mdpi
72x72 hdpi
96x96 xhdpi
144x144 xxhdpi
192x192 xxxhdpi

for each dpi size, we need to specify 1 specific image, or use svg

e.g: if app requires 16px to detect a fling action:
-16/160dpi = 1/10th of an inch, approx. 2.5mm
-16/240dpi = 1/15th of an inch, approx. 1.7mm
*notice the difference on dpi and actual swipeable area

With layout_weight you can specify a size ratio between multiple views. E.g. you have a MapView and a table which should show some additional information to the map. The map should use 3/4 of the screen and table should use 1/4 of the screen. Then you will set the layout_weight of the map to 3 and the layout_weight of the table to 1.

To get it work you also have to set the height or width (depending on your orientation) to 0px.
